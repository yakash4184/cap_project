Civic Issue Reporting and Management System

Project Type: Full-stack civic issue reporting platform

Frontend: Next.js, React, Tailwind CSS

Backend: Next.js Route Handlers, MongoDB, Mongoose

Authentication: JWT-based auth with Citizen and Admin roles

Current Local URL: App UI + API on localhost:3000

How To Use The Platform

  1. Open the frontend on http://localhost:3000.
  2. Create an account from the Register page.
  3. Login using your email and password.
  4. Citizens can report issues from the Citizen Desk.
  5. Admins can manage all issues from the Admin Control dashboard.

Features That Need Extra Setup

  • Real Cloudinary image upload requires Cloudinary environment variables
  • Live Google Maps markers require a Google Maps API key
  • Production cron behavior requires Vercel cron and env setup
  • Production realtime verification should be tested after deployment

Deployment Notes

  • Deploy single app from the client folder on Vercel
  • Set all env vars from client/.env.example in Vercel
  • Keep NEXT_PUBLIC_API_BASE_URL set to /api
  • Use MongoDB Atlas for the production database
